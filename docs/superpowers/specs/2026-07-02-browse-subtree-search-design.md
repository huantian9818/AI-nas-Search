# 目录页当前目录搜索设计

## 目标

在目录页增加“当前目录搜索”能力。搜索范围是当前目录及其全部子目录，结果直接在目录页右侧内容区展示，不跳转到全局搜索页。

这次实现要保持目录页现有的核心体验：

- 左侧共享目录树不变。
- 右侧仍然使用现有卡片网格风格。
- 图片继续复用现有缩略图缓存链路。
- 文件继续复用现有下载能力。
- 点击目录结果进入目录；点击文件结果进入其父目录并高亮该文件。

## 推荐方案

推荐在 `/browse` 内直接增加一条“限定当前目录子树”的搜索链路，而不是复用全局 `/search` 页面。

原因：

- 目录页的上下文已经是“当前路径”，用户不需要切换页面。
- 搜索范围天然依附于当前 `path`，放在 `/browse` 里最清晰。
- 结果展示需要沿用目录页卡片网格、缩略图、下载和高亮定位，这些能力本来就都在目录页。
- 避免把目录页和全局搜索页的状态、模板和参数耦合在一起。

## 路由与参数

`/browse` 新增可选查询参数：

- `path`：当前目录路径，沿用现有行为。
- `q`：当前目录搜索关键词。
- `selected`：沿用现有文件高亮定位行为。
- `page`：普通目录浏览分页，当前目录搜索模式下不再使用分页。

行为规则：

- `q` 为空或全空白：按现在的普通目录浏览逻辑渲染。
- `q` 非空：右侧切换到“当前目录搜索结果”模式。
- 搜索结果里的目录链接仍然进入 `/browse?path=<directory>`。
- 搜索结果里的文件链接进入 `/browse?path=<parent>&selected=<id>`。
- 清空搜索后回到当前目录普通浏览视图。

## 查询范围与排序

当前目录搜索只查当前路径子树，不查整个 NAS，也不查其他共享目录。

查询条件：

- `nas_id` 必须等于当前登录 NAS。
- `share_path` 必须属于当前用户可访问共享目录。
- `full_path` 必须满足：
  - 当前路径为 `/` 时，允许所有当前用户可访问的共享目录结果。
  - 当前路径不为 `/` 时，只允许 `full_path = path` 或 `full_path LIKE path + '/%'`。
- `name` 必须命中关键词。

排序建议沿用现有搜索习惯：

1. 目录在前，文件在后。
2. 同类结果按 `lower(name)`。
3. 再按 `id` 稳定排序。

第一版不做独立分页，直接返回该子树内的完整结果集合。这样用户能一次看到当前目录下的全部命中项，符合“当前目录搜索”的预期。

## 仓储接口

在 `EntryRepository` 中新增一条专用于目录页的查询方法：

- `search_subtree(...)`

职责：

- 输入 `nas_id`、当前 `path`、`query`、`allowed_share_paths`。
- 返回当前目录子树内全部命中项。
- 输出类型继续复用 `Page[Entry]`，但搜索模式下固定为单页：
  - `page = 1`
  - `page_size = len(rows)`
  - `total = len(rows)`

实现策略：

- 关键词长度小于 3 时，沿用当前 `ILIKE` 模糊匹配逻辑。
- 关键词长度大于等于 3 时，优先沿用现有 FTS 查询路径，再额外叠加 `full_path` 范围约束。

这样可以最大化复用当前搜索能力，同时把范围限制清楚地压在 SQL 层，而不是查出来后再用 Python 二次过滤。

## 页面交互

目录页右侧在“当前路径”下面增加一个紧凑搜索区域：

- 标签文案：`在当前目录及子目录搜索`
- 输入框：输入关键词
- 按钮：`搜索`
- 有关键词时额外显示 `清空`

右侧内容区分成两种模式：

### 普通浏览模式

沿用当前行为：

- 显示当前目录直接子项数量。
- 显示现有卡片网格。
- 保留当前批量下载、多选、全选逻辑。
- 保留普通浏览分页。

### 当前目录搜索模式

右侧切成“搜索结果网格”：

- 顶部显示：当前目录搜索命中数量。
- 不显示普通浏览的“共 N 项”语义。
- 不显示普通浏览分页。
- 不显示批量下载区，避免与“跨子目录结果”冲突。
- 结果卡片仍使用现有 `browse-tile` 风格，减少视觉跳变。

每个结果卡片新增一行相对路径说明：

- 目录结果：显示相对当前目录的目录路径。
- 文件结果：显示相对当前目录的父目录路径。
- 当前路径本身命中时，可显示 `/` 或“当前目录”。

结果卡片行为：

- 目录卡片：点击进入该目录。
- 文件卡片：
  - 卡片主体可点击进入父目录并高亮该文件。
  - 缩略图照旧可显示。
  - 下载按钮照旧复用已有 `/downloads/{id}`。

## 结果路径展示

新增一个“相对当前目录路径”格式化逻辑，用于目录页搜索模式：

- 当前路径 `/设计部/包装设计`
- 命中文件 `/设计部/包装设计/速食线/方案A/苹果图.png`
- 显示相对路径 `速食线/方案A`

规则：

- 目录结果显示目录本身相对路径。
- 文件结果显示其父目录相对路径。
- 若相对路径为空，显示 `当前目录`。

这样用户能快速知道命中项分布在哪个子目录层级，而不需要先点进去再判断。

## 错误处理与空状态

- `q` 为空：回到普通目录浏览，不报错。
- `q` 有值但没有命中：右侧显示“当前目录及子目录下没有匹配项”。
- `path` 非法或无权限：沿用当前目录页的访问控制和 404/重定向逻辑。
- 用户刷新搜索结果页：根据 URL 中的 `path` 和 `q` 直接恢复当前目录搜索结果。

## 性能边界

当前目录搜索的性能取决于当前路径子树大小，但通常仍然比全局搜索更轻，因为额外加了明确的 `full_path` 范围约束。

第一版不做以下优化：

- 不单独维护目录子树索引表。
- 不引入搜索结果缓存。
- 不为当前目录搜索增加独立 AI 能力。

如果后续发现某些超大目录子树搜索仍偏慢，再评估专门的子树索引或路径前缀辅助索引。

## 测试

需要补充三类测试：

### 目录页路由测试

- `q` 为空时仍走普通目录浏览。
- `q` 有值时只返回当前目录子树命中项。
- 不应返回当前目录外的兄弟目录或其他共享目录结果。

### 页面渲染测试

- 搜索模式显示搜索输入区、命中数量和结果卡片。
- 搜索模式显示相对路径。
- 搜索模式下目录和文件链接分别指向正确的 `/browse` URL。
- 搜索模式下文件卡片仍保留缩略图和下载按钮。
- 无结果时显示新的空状态文案。

### 交互回归测试

- 普通目录浏览模式下现有批量下载和分页行为不变。
- 搜索模式下不渲染普通分页和批量下载区。
- 清空搜索后恢复普通目录浏览内容。
